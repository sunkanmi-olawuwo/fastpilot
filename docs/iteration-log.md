## Week 1‚Äì4 (the progression that set the production config)
- **Week 1 ‚Äî corpus + naive baseline.** Assembled the **514-document** corpus across 4 sources
  (`official_docs` 156, `github_issue` 200, `github_discussion` 100, `template_repo` 58). A naive
  word-splitter + BGE-large (512-token window) **silently truncated 27.9% of chunks (963/3,455)** ‚Äî
  a quarter of the index lost content before it was ever embedded (GitHub issues run ~1,688-word
  median, ~10√ó BGE's limit). This finding drove the Week-2 chunker + embedder rework.
- **Week 2 ‚Äî chunking + the embedder migration.** Built the language-routed hybrid chunker (AST
  tree-sitter for code, markdown-aware recursive for prose) and **migrated BGE ‚Üí Voyage-4-lite
  (2048-d, 32K-token window)**. Result: **truncation 27.9% ‚Üí 0** across all **4,232** hybrid chunks
  (a BGE re-run on the same chunks would still truncate 9.3% ‚Äî the embedder swap, not just the
  chunker, closes the gap). Dense-only baseline scored 24 pts.
- **Week 3 ‚Äî retrieval head-to-head.** Ran a 4-way pairwise eval (72 judge calls, 12 hold-out
  questions). **T1b (hybrid ‚Üí RRF ‚Üí Voyage rerank-2.5 ‚Üí top 10) won 30/36 comparisons**, 8/12
  first-place finishes, never last. Reranking was the single biggest lever: **T1b 43 vs T1a 22 =
  +21 pts**. **T3 (two-stage LLM file routing) was evaluated and deliberately skipped** ‚Äî competitive
  on retrieval but **~34s/query** (~3.3√ó T1b's latency). Honest split recorded: T1b wins answer
  quality, T3 wins exact-file retrieval ‚Üí motivates the Week-5 query-shape router. (A 5-way extension
  confirmed T1b at 52 vs class-style T2 at 39.)
- **Week 4 ‚Äî evaluation instrument.** Built the triangulated eval (decomposed Gemini faithfulness
  judge, iterated v1‚Üív3; deterministic Voyage answer-coverage). Offline T1b baseline:
  **faithfulness 0.952 / coverage 0.941** ‚Äî the numbers the Week-5 production-parity run is measured
  against. Full rationale in [`evaluation-strategy.md`](evaluation-strategy.md).

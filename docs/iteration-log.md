- 2026-06-11 — Live verification surfaced a real data bug: the configured production collection `week3_hybrid` carried only `{category}` per-chunk meta — `file_path` was gone. Root cause: week-3's `04_apply_categories_to_chunks.py` did `set_payload(payload={"meta": {"category": …}})`, which *replaced* the whole `meta` object (wiping `file_path`/`source`/`title`) instead of merging; vectors were untouched. Since the sources panel, the Tier-D exact-file dogfood probes, and the Week-4 `expected_file_hit` story all need `file_path`, re-indexed a clean production collection `rag_accelerator_capstone_final` via `scripts/07_reindex_production.py`: reuses the proven week-3 hybrid pipeline (AST chunker, Voyage-4-lite dense, Qdrant/BM25 sparse, `text-dense`/`text-sparse` named vectors) but attaches `file_path` + `category` (from `category_map.json`) to each doc's meta *before* chunking so both propagate. Result: 4,232 chunks (exact match), 514/514 docs categorised, full meta (`file_path`, `category`, `source`, `title`, `url`). Repointed `QDRANT_COLLECTION`; live retrieval now shows real sources (`official_docs::tutorial/security/simple-oauth2/index.md`, template `README.md`, issue/discussion IDs). Also added a resilient `app/formatting.source_label` (falls through file_path → file → title → source_id → source → category) so the panel is never blank regardless of collection.
- 2026-06-11 — Latency work after live testing (cross-region dev box inflated everything): (1) prompt registry now caches fetched prompts 60s so `build_prompt` stops making a per-request Opik HTTP call; (2) **embed-reuse** — verified the cache-lookup vector and the retrieval query vector are *identical* (voyage-4-lite, 2048d, input_type=None → cosine 1.000000), so a cache miss now embeds the query **once** (in `cache.get`) and reuses that vector for retrieval and `cache.set` instead of 3× . Implemented by keeping the dense embedder out of the Haystack graph and supplying `retriever.query_embedding` directly; live-validated that retrieval is byte-identical (same doc ids + scores) with and without the reused embedding. Saves 2 Voyage calls per miss (cost + latency; the win is masked by cross-region RTT here but real once co-located). The remaining latency levers (local classifier, co-location) are Phase 4/5.
